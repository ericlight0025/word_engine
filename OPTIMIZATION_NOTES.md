# Word 版型合併工具優化清單

日期：2026-04-05

## 目前建議優先順序

### 1. 未儲存變更提醒

目的：
- 避免使用者修改資料後忘記存回原始檔

建議行為：
- 任一欄位被修改後，標示 `未儲存變更`
- 切換資料列、切換檔案、切換頁面、關閉程式前先提示
- 提供：
  - `存檔後繼續`
  - `不存檔繼續`
  - `取消`

### 2. 單筆套版結果預覽

目的：
- 讓右側不只是欄位對照，而是更接近實際輸出內容

建議行為：
- 使用目前選取的資料列
- 顯示這一筆資料套進模板後的關鍵段落
- 至少先做文字預覽，不必一開始就做完整 Word 視圖

### 3. 欄名直接編輯

目的：
- 讓使用者能快速客製化資料表欄名

建議行為：
- 表頭可雙擊改名
- 改名後同步更新：
  - 工作台資料欄位
  - 下方完整欄位編輯區
  - Tag 對照預覽
- 可存回 CSV / Excel

### 4. 批次輸出前確認頁

目的：
- 降低批次產出前的操作錯誤

建議行為：
- 顯示：
  - 已選資料筆數
  - 目前模板名稱
  - 輸出資料夾
  - 缺欄位警示
  - 空值欄位統計
- 使用者確認後才開始輸出

### 5. 命名規則模板化

目的：
- 讓輸出檔名更符合實際工作需求

建議行為：
- 支援輸出命名格式，例如：
  - `{{ 合約編號 }}`
  - `{{ 合約編號 }}_{{ 姓名 }}`
  - `{{ 日期 }}_{{ 公司名稱 }}`
- 對非法檔名字元自動清理

### 6. 設定頁記住最後選取檔案

目的：
- 減少每次重開都要重新切換資料檔和模板

建議行為：
- 除了記住資料夾，也記住：
  - 上次選取的 CSV / Excel 檔
  - 上次選取的 Word 模板檔
  - 上次使用的主題與字型倍率

### 7. Windows 實機驗證

目的：
- 確保未來在 Windows 交付時不出現平台問題

建議驗證項目：
- `customtkinter` 顯示是否正常
- 路徑選擇與存檔是否正常
- `.doc` 轉換是否可透過 LibreOffice 正常執行
- 產出完成後開啟資料夾功能是否正常

### 8. 表格互動細節優化

目的：
- 提升大量資料操作效率

建議行為：
- 表格 hover 深色高亮更穩定
- 排序方向圖示更明顯
- 支援欄寬調整
- 支援鍵盤上下切換資料列

### 9. 產出記錄追蹤

目的：
- 讓批次產出後可以追蹤每次輸出的時間、來源、模板與結果
- 降低重複產出、檔案難以追查、錯誤難以回溯的問題

建議行為：
- 每次按 `批次產出` 都建立一筆批次記錄
- 至少記錄：
  - 產出時間
  - 資料來源檔名
  - 模板檔名
  - 輸出資料夾
  - 輸出筆數
  - 成功 / 失敗狀態
  - 使用的檔名欄位
  - 每一筆產出的檔名
- 若有失敗，保留失敗原因摘要

建議實作範圍：
- 第一階段先做本地記錄
  - `output_history.jsonl` 或 `history.sqlite`
- 第二階段再考慮新增 UI 的 `產出記錄` 查詢頁
- 目前先列入未來規劃，不在這一版實作

## 我最建議先做的項目

第一優先：
- 未儲存變更提醒

原因：
- 這個最直接降低資料遺失風險
- 比純視覺優化更影響實際使用

第二優先：
- 單筆套版結果預覽

原因：
- 可以讓 `目前資料套版預覽` 真正變得有價值

第三優先：
- 欄名直接編輯

原因：
- 這會讓整套工具更接近實際商務作業流程

後續可列入下一階段：
- 產出記錄追蹤

原因：
- 一旦開始正式產出文件，就會需要基本的追蹤與回查能力
