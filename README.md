> 從資料表到 Word 文件，不再手動複製貼上，而是直接套版、檢查、輸出。

## 一句話介紹

這不是單純的 Word merge 腳本，而是一個把 `資料匯入`、`欄位確認`、`內容修改`、`套版預覽`、`批次輸出` 整理成同一套桌面操作流程的文件工作台。

## 核心價值

- 用 Excel / CSV 直接驅動 Word 文件產出，不再手動逐份複製貼上。
- 在輸出前就能先檢查資料與 Tag 對應結果，降低文件錯誤率。
- 把重複文件處理流程固定化，讓非工程背景的使用者也能穩定操作。

## 這個工具解決什麼問題

很多行政、業務、顧問、設計或教育場景，都有一種很耗時間的工作流程：資料其實早就在 Excel 或 CSV 裡，但文件還是得一份一份手動貼進 Word，改姓名、日期、金額、公司名稱、合約編號，再另存成不同檔案。

這個工具就是拿來處理這段重複流程。你準備好資料表與 Word 模板後，它會自動對應 `{{ 欄位名稱 }}` Tag，讓你先在畫面上檢查內容，再一次批次產出整批文件。

## 這個工具和一般 Mail Merge 有什麼不同

很多 Mail Merge 方案本質上是函式庫、腳本或外掛，重點放在「把資料塞進文件」。這些方案對開發者很好用，但對一般使用者來說，通常還是得自己處理資料整理、欄位確認、預覽檢查、檔名管理和批次輸出。

這個工具的定位不是單純的文件替換引擎，而是一個可以直接操作的桌面工作台。它把 `資料匯入`、`欄位檢查`、`資料修改`、`套版預覽`、`批次輸出` 放進同一個介面裡，讓非工程背景的使用者也能完成整套文件產出流程。

換句話說，一般 Mail Merge 工具解決的是：

- 我可以把資料套進 Word。

這個工具解決的是：

- 我可以把整份 Excel / CSV 名單，變成一套可檢查、可修改、可批次輸出的文件流程。

## 解決點

- 減少手動複製貼上 Word 內容的時間成本。
- 降低貼錯欄位、漏改內容、覆蓋舊資料的風險。
- 在輸出前先預覽套版結果，而不是輸出後才發現錯誤。
- 可以直接在工具內修正資料與欄名，不用在 Excel 和 Word 之間反覆切換。
- 把重複文件處理流程固定化，適合日常大量產出。

## 目前支援

- 匯入 `.csv` / `.xlsx` / `.xls`
- 選擇 `.docx` / `.doc` 模板
- 掃描 `{{ 欄位名稱 }}` Tag
- 批次輸出多份 Word 文件
- 工作台直接編輯資料內容
- 自訂欄名並存回來源檔
- 三頁式介面：
  - `主工作區`
  - `設定`
  - `說明`
- 5 組深色主題
- 字型倍率調整
- 批次完成後自動開啟輸出資料夾

## 快速開始

第一次使用時，建議照下面順序操作：

1. 開啟程式後，先切到 `設定` 頁。
2. 分別指定 `CSV / Excel 資料夾`、`Word 範本資料夾`、`輸出資料夾`。
3. 從 `資料檔` 下拉選單選擇要匯入的 `.csv`、`.xlsx` 或 `.xls`。
4. 從 `Word 範本` 下拉選單選擇要套用的 `.docx` 或 `.doc` 模板。
5. 按 `套用設定`，讓目前介面載入剛剛選定的資料與模板。
6. 回到 `主工作區`，在左側表格確認資料是否正確。
7. 如果需要修正內容，可直接雙擊前 6 欄儲存格，或在下方 `完整欄位編輯` 修改完整資料。
8. 如果要調整欄位名稱，可在 `欄名客製化` 區輸入新名稱後按 `儲存欄名`。
9. 修改完成後，按 `存回表格` 或 `存回原始檔`，把資料變更保存下來。
10. 在右側 `目前資料套版預覽` 檢查 Tag 是否有正確對應到資料內容。
11. 在上方 `檔名欄位` 指定輸出檔名要使用哪一個欄位。
12. 選取要輸出的資料列後按 `批次產出`，完成後程式會自動開啟輸出資料夾。

使用重點：

- 如果沒有手動選取資料列，可先按 `全選資料` 再執行 `批次產出`。
- 模板中的 Tag 必須和資料欄名一致，例如 `{{ 姓名 }}` 會對應到欄名 `姓名`。
- 處理 `.doc` 模板時，系統需要 LibreOffice 協助轉換。

## 圖文操作說明

### 主工作區

這個畫面是你最常使用的工作區，主要負責選資料、改內容、預覽結果、批次輸出。

主要區塊：

- `主工作區`
- `檔名欄位`
- `全選資料`
- `清除選取`
- `批次產出`
- `Excel 資料區`
- `欄名客製化`
- `完整欄位編輯`
- `目前資料套版預覽`

操作方式：

1. 在左側 `Excel 資料區` 選擇要處理的資料列。
2. 如果只要處理部分資料，可手動選取；若要全部處理，按 `全選資料`。
3. 前 6 欄可直接雙擊儲存格修改內容。
4. 所有欄位都可在 `完整欄位編輯` 區修改。
5. 如果要調整表頭名稱，在 `欄名客製化` 區輸入後按 `儲存欄名`。
6. 修改內容後按 `存回表格` 或 `存回原始檔`。
7. 在右側 `目前資料套版預覽` 檢查這一筆資料對應到模板後的欄位狀態。
8. 在上方 `檔名欄位` 選擇輸出檔名要使用的欄位。
9. 按 `批次產出` 產生文件。

### 設定

`設定` 頁負責指定來源資料與輸出位置。第一次使用時，建議先完成這一頁再回到主工作區。

設定項目：

- `資料夾設定`
- `CSV / Excel 資料夾`
- `Word 範本資料夾`
- `輸出資料夾`
- `主題`
- `字型倍率`
- `資料檔`
- `Word 範本`
- `重新掃描資料夾`
- `儲存設定`
- `套用設定`

操作方式：

1. 設定 `CSV / Excel 資料夾`。
2. 設定 `Word 範本資料夾`。
3. 設定 `輸出資料夾`。
4. 從 `資料檔` 下拉選單指定目前使用的 CSV / Excel。
5. 從 `Word 範本` 下拉選單指定目前使用的模板。
6. 需要重讀資料夾時按 `重新掃描資料夾`。
7. 要保留目前設定時按 `儲存設定`。
8. 要立即套用設定到目前介面時按 `套用設定`。
9. `主題` 可切換 5 組深色風格。
10. `字型倍率` 可調整整體介面大小。

`說明` 頁主要是快速導覽，不負責編輯資料，但可以幫助新使用者了解目前這份專案設定的用途。

內容包含：

- `說明`
- `顧問合約與付款通知批次產出`
- 案例說明文字
- 目前資料來源摘要

用途：

- 查看這份工具目前預載的案例用途
- 查看目前資料來源與模板摘要
- 當成新使用者的快速導覽頁

## 介面重點

- 左側 `Excel 資料區` 用來選取和檢查每一筆資料。
- 下方 `完整欄位編輯` 可修改目前選取資料列的完整欄位內容。
- `欄名客製化` 可重新命名欄位並同步更新資料來源。
- 右側 `目前資料套版預覽` 可先確認 Tag 對應結果，再決定是否輸出。
- 上方 `檔名欄位`、`全選資料`、`批次產出` 是主要操作控制區。

## 資料編輯

### 編輯欄位內容

- 表格前 6 欄：
  可雙擊儲存格直接修改
- 所有欄位：
  可在下方 `完整欄位編輯` 區修改

### 編輯欄名

- 在 `欄名客製化` 區輸入新欄名
- 按 `儲存欄名`
- 會同步更新：
  - 目前資料表
  - 下方完整欄位編輯區
  - Tag 對照
  - 原始 `.csv` / `.xlsx`

限制：

- 欄名不可空白
- 欄名不可重複

## Tag 規則

Word 模板中的 Tag 格式：

```txt
{{ 欄位名稱 }}
```

例如：

```txt
姓名：{{ 姓名 }}
日期：{{ 日期 }}
合約編號：{{ 合約編號 }}
```

比對規則：

- 欄名需完全一致
- 大小寫與空白差異都算不同

右側預覽狀態：

- `✅` 找到對應欄位
- `⚠️` 模板有 Tag，但資料沒有對應欄位
- `❌` 資料有欄位，但模板沒有使用

## `.doc` 轉換

`.doc` 會先透過 LibreOffice CLI 轉成暫存 `.docx` 再處理。

需求：

- 系統已安裝 LibreOffice
- 命令列可執行 `soffice`

原始 `.doc` 不會被覆蓋。

## 設定檔與路徑策略

目前專案採用以下原則：

- 內建 demo 資源放在專案內
- 使用者資料夾路徑由設定頁指定
- 輸出資料夾可自訂
- `.doc` 轉換暫存檔走系統 temp

`settings.json` 屬於本機設定，不建議提交到 Git。
